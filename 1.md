# Пошаговая инструкция по развертыванию VHS Restorer на RunPod

bash <(curl -sL 'https://raw.githubusercontent.com/Dizzle700/LTX/main/server_bootstrap.sh') # Не работает в RunPod из-за ограничений оболочки
```

Вместо этого используйте стандартный пайп через `|`:

```bash
bash -c "curl -sL https://raw.githubusercontent.com/Dizzle700/LTX/main/server_bootstrap.sh | bash"
```

Или универсальный вариант (если `curl` не установлен, сработает через `wget`):
```bash
bash -c "(curl -sL https://raw.githubusercontent.com/Dizzle700/LTX/main/server_bootstrap.sh || wget -qO- https://raw.githubusercontent.com/Dizzle700/LTX/main/server_bootstrap.sh) | bash"
```

Этот чек-лист поможет вам настроить и запустить серверную часть проекта для реставрации видео на RunPod, а также подключить к нему клиентское приложение.

---

## Шаг 1. Подготовка ссылки (однократно при изменении кода)

Убедитесь, что ваш скрипт запуска [server_bootstrap.sh](file:///home/snipe/Code/LTX/server_bootstrap.sh) загружен на GitHub (или в GitHub Gist), и вы скопировали прямую (Raw) ссылку на него.

* **Пример ссылки (GitHub):**  
  `https://raw.githubusercontent.com/username/vhs-restorer-runpod/main/server_bootstrap.sh`
* **Пример ссылки (Gist):**  
  `https://gist.githubusercontent.com/username/gist_id/raw/server_bootstrap.sh`

## Шаг 2. Создание и настройка пода в RunPod

1. Авторизуйтесь на сайте [RunPod.io](https://runpod.io) и перейдите во вкладку **GPU Cloud**.
2. Выберите подходящую видеокарту (минимум 16 ГБ VRAM, рекомендуется **RTX 4090 (24GB)** или **A100**) и нажмите кнопку **Deploy**.
3. В открывшемся окне выбора шаблона (Template) выберите стандартный шаблон:  
   `RunPod PyTorch 2.x` (под капотом Ubuntu 22.04 и CUDA 12.x).
4. **⚠️ КРИТИЧЕСКИ ВАЖНО:** Перед тем как нажать финальную кнопку Deploy, нажмите на кнопку **Customize Deployment** (Настроить контейнер):
   * В поле **Container Start Command** вставьте команду для загрузки и выполнения вашего скрипта:
     ```bash
     bash -c "curl -sL ВАША_RAW_ССЫЛКА_НА_GITHUB_ЗДЕСЬ | bash"
     ```
     *(Или универсальный вариант: `bash -c "(curl -sL ССЫЛКА || wget -qO- ССЫЛКА) | bash"`)*
   * В поле **Expose Ports** (или HTTP Ports) введите порт **8000** (на этом порту будет работать FastAPI сервер).
5. Нажмите **Deploy** (или *Continue* -> *Deploy*).

---

## Шаг 3. Ожидание установки (5–15 минут)

После перевода пода в статус `Running` запустится скрипт инициализации. Он автоматически обновит пакеты, склонирует репозиторий, настроит Python-окружение и скачает около 20 ГБ весов моделей.

* **Как следить за прогрессом:**
  1. В списке подов наведите на ваш под и нажмите кнопку **Logs** -> **System Logs**.
  2. *Альтернативно:* Подключитесь к поду через **Web Terminal** и выполните команду для просмотра лога установки:
     ```bash
     tail -f /workspace/bootstrap.log
     ```
* **Завершение установки:**
  Установка завершена, когда в логе `bootstrap.log` появится строка:
  ```
  [OK] Server is UP! PID=...  Port=8000
  ```

## Шаг 4. Подключение через клиентское приложение

1. В списке подов RunPod нажмите кнопку **Connect** напротив вашего пода.
2. Найдите порт **8000** и скопируйте прокси-URL. Он выглядит следующим образом:
   ```
   https://xxxxxxxxxxxxxxxx-8000.proxy.runpod.net
   ```
3. Запустите клиентское приложение на локальном компьютере:
   ```bash
   python vhs_restorer_client.py
   ```
4. Вставьте скопированный прокси-URL в поле **Server URL** и нажмите кнопку **Connect**.
5. Проверьте статус подключения: в клиенте должна отобразиться информация о вашей GPU и свободной видеопамяти (VRAM), а индикатор станет зеленым: **⬤ Connected**.

*Теперь вы можете перетащить VHS-видео в drop-зону клиента, выбрать настройки и нажать **Start Processing**!*
